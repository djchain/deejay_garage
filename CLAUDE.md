# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库性质

这是 Deejay 的个人实验场（Garage）—— 没有固定主题或统一技术栈。每个子目录可能是一个独立的小项目、代码片段、玩具或实验，互不依赖。

## 工作原则

1. **先探索，再行动**：这个仓库没有统一的项目结构。进入任何子目录后，先列出文件、阅读相关代码、理解该项目自身的约定，再动手修改。
2. **尊重子项目的独立性**：不要假设一个子目录的依赖、配置或约定适用于另一个。
3. **轻量优先**：Garage 的本意就是随手放、随手改。不用为小实验搭建重型框架，除非该项目本身需要。
4. **有疑问就问**：如果某个子目录的意图不清晰，向 Deejay 确认而非猜测。

## 目录约定（推荐，非强制）

- 每个实验/玩具项目放在自己的子目录下
- 如果某个子项目有自己独特的构建/测试/开发流程，建议在该子目录下也放一个 `CLAUDE.md` 记录之
